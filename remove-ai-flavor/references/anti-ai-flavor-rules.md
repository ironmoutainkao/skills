# 去 AI 味规则表

## 识别特征

| 类型 | 典型表现 | 处理方向 |
|------|----------|----------|
| 套路开场 | 在这个快速发展的时代、随着 AI 的发展、众所周知 | 删除铺垫，直接进入问题或场景 |
| 生硬连接 | 综上所述、基于此、由此可见、鉴于上述 | 改成自然承接，或直接删除 |
| 空洞大词 | 卓越、革命性、颠覆性、重要意义、巨大价值 | 用事实、数据、场景、结果替代 |
| 假金句 | 句子很响，但没有新信息 | 删掉，或改成具体判断 |
| 破折号滥用 | A——B、观点——解释、概念——定义 | 改用冒号、逗号、句号、括号，或重排句式 |
| 模板对立句 | 不是 A，而是 B | 直接说 B，或写成递进关系 |
| 复杂排比 | 不仅 A，而且 B，更 C | 拆成短句，保留真正重要的信息 |
| bullet 堆叠 | 每段都是列表，像 AI 总结 | 必要列表保留，其余改成自然段 |
| 乱加粗 | 每段都有加粗，重点过多 | 只保留真正影响理解的关键词 |
| 翻译腔 | 语序像英文，抽象名词多，动作词少 | 用自然中文重说，重心前置 |
| 判词腔 | 用“不是免费的 X 来源”“必须看起来 Y”这类负向定义、制度化判断表达观点 | 改成更直接的判断、动作句或带人味的说法 |
| 资料味 | 元信息、背景、定义堆在开头 | 元信息后置，正文先开讲 |
| 讲义味 | 每段像定义，标题过于整齐 | 加入阅读牵引和轻微节奏变化 |

## 高频替换

| AI 味表达 | 更自然的处理 |
|-----------|--------------|
| 综上所述 | 所以 / 总的来说 / 直接说结论 |
| 鉴于此 | 因为这样 / 这也意味着 |
| 不难发现 | 删除，直接陈述 |
| 值得一提的是 | 删除，直接说 |
| 众所周知 | 删除，或补具体证据 |
| 具有重要意义 | 具体说明“对谁有什么用” |
| 这无疑是 | 删除“无疑”，用事实支撑 |
| 在这个快速发展的时代 | 直接进入具体问题 |
| 随着技术不断发展 | 说明具体变化是什么 |
| 让我们一起 | 改成具体行动或直接收束 |

## 负向定义和判词腔

有些句子问题不在套话，而在“像模型下判词”：先用否定结构框住概念，再给一个抽象名词。它可能逻辑正确，但不像真人会讲。

典型特征：

- `不是 + 抽象名词/来源/机制/保证`
- `必须/需要 + 看起来/表现为/呈现出 + 抽象关系`
- `不能简单理解为/不应被视为/并非单纯的`
- 名词很多，动作很少，句子像制度说明或风控报告

处理方向：

- 能正着说就正着说，不必保留原来的否定框架。
- 能用动作说清楚，就少用“来源、机制、保证、解释、呈现”这类抽象名词。
- 如果原文有锋利判断，可以保留锋利，不要一律改温和。
- 如果原文是专业语境，保留必要限定，但把句子重心放到读者能立刻懂的判断上。

| 原表达 | 问题 | 更自然的改法 |
|--------|------|--------------|
| 它们不是免费的可靠性来源 | “不是免费的 X 来源”像翻译腔和判词腔，抽象、绕 | 可靠性没有那么简单 / 别把它们当成白来的可靠性 |
| 但它对外的解释必须看起来合作 | “解释必须看起来合作”很制度化，动作不清楚 | 但它对外还要装作是合作的结果 / 但对外还得说成是双方合作出来的 |
| 这并不是一个单纯的效率问题 | 模板否定句，信息重心太晚 | 问题不只在效率 / 真正麻烦的不是效率 |
| 它不应被视为长期稳定性的保证 | 被动、抽象，像免责声明 | 别指望它长期稳定 / 它撑不起长期稳定 |

## 破折号替代

优先避免 `——`。可按语义替换：

- 解释关系：`A——B` -> `A，也就是 B` / `A：B`
- 补充说明：`A——尤其是 B` -> `A，尤其是 B`
- 强转折：拆成两句
- 定义关系：`A——一种 B` -> `A 是一种 B`

## 资料稿改文章

当原稿像资料、笔记、提纲或翻译稿时，按这个顺序处理：

1. 元信息后置：原文链接、作者、发布时间、来源不要挡在开头。
2. 开头先开讲：用一个真实问题、具体场景或作者判断引出主题。
3. 小标题文章化：不要全是“定义、特点、方法、总结”。
4. 段间补桥：让读者知道为什么从 A 讲到 B。
5. 结尾收回主判断：不要只用清单或“以上就是”结束。

## 句子改写法

优先使用这几种手法：

- 删：删掉没有信息量的套话、形容词、连接词。
- 拆：长句拆成两三句。
- 换：抽象词换成具体动作、对象、场景。
- 顺：把句子重心提前，先说结论，再补解释。
- 接：补一句自然过渡，让段落之间有关系。

## 风格边界

不要这样改：

- 不要为了像人写，加入夸张情绪。
- 不要为了顺滑，把事实和条件删空。
- 不要把技术文章改成鸡汤。
- 不要把克制判断改成营销腔。
- 不要改变作者原本立场。

## 最终质感

好的去 AI 味，不是把文章改得更花，而是让它更像一个具体的人在讲清楚一件具体的事：

- 有观点，但不端着。
- 有节奏，但不堆技巧。
- 有温度，但不油。
- 有信息密度，但读起来不费劲。
