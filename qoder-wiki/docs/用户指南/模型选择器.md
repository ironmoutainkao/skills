# 模型选择器

来源: https://docs.qoder.com/zh/user-guide/chat/model-tier-selector

## 概述

Qoder 提供两类模型选择方式：

- 分级选择：按能力和成本选择模型池
- 指定模型：直接选择某个具体模型
- 自定义模型：通过 API Key 接入第三方模型，详见 `自定义模型.md`

## 分级选择

| 级别 | 说明 | 适用场景 | Credits 消耗 |
| --- | --- | --- | --- |
| Auto | 智能路由，平衡性能与成本 | 日常开发默认选择 | 约 1.0x |
| Ultimate | 最强推理与深度思考 | 复杂系统设计、疑难问题 | 约 1.6x |
| Performance | 高质量输出 | 核心功能实现、重构 | 约 1.1x |
| Efficient | 高性价比 | 基础生成、测试、普通问答 | 约 0.3x |
| Lite | 免费基础模式 | 快速验证、轻量问答 | 免费 |

## 使用建议

- 默认优先用 `Auto`
- 高价值任务可切到 `Ultimate` 或 `Performance`
- 成本敏感或简单任务用 `Efficient`
- 临时尝试和快问快答可用 `Lite`

## 常见说明

- 模型列表可能随官方策略更新
- 同一会话中可以切换模型
- 分级选择偏“按档位选”，指定模型偏“按具体型号选”
- 不同模型的 Credits 消耗系数不同
