# 智能会话概述

Qoder 提供两种模式：智能问答和智能体，这些能力可帮助开发者解决编码问题、修复错误、调试并排查运行时问题。Qoder 还支持多文件编辑、自主决策、环境感知与工具调用，助力完成端到端的编码任务。

## 核心功能

### 1. 多种聊天模式

在同一对话流程中，开发者可在智能问答和智能体模式间自由切换。这种灵活性提升了各类开发工作流的生产力与效率。

### 2. 自动环境感知

Qoder 会自动从任务描述中识别项目框架、技术栈、所需代码文件与错误信息。这样无需手动补充上下文，使任务描述更为简洁。

### 3. 工具使用

Qoder 可自主调用 10+ 款内置工具，辅助进行文件读写、代码查询与错误排查。它还支持 MCP（Model Context Protocol）工具配置，允许开发者按需定制工具集。

### 4. 命令执行

Qoder 可自主判断、生成并执行必要的命令，显著提升任务执行效率。

### 5. 项目级变更

基于任务描述，Qoder 支持修改项目中的多个代码文件。通过多轮对话，可进行代码优化或快照回滚，更高效地完成任务。

### 6. 记忆感知

Qoder 具备基于 LLM 的自主记忆能力。它会从每次对话中学习，逐步构建与个人开发者、特定项目及已遇问题相关的丰富记忆库。

## 开始新聊天

### 打开智能会话面板

要开始 AI 对话，登录 Qoder，然后在右上角切换次级侧边栏。

或者使用键盘快捷键：

| 操作 | macOS | Windows |
| --- | --- | --- |
| 打开/关闭智能会话面板 | `⌘` `L` | `Ctrl` `L` |

### 选择模式

- **智能问答**：一个简单的问答模式，用于解答编程问题。它会基于上下文提供解决方案和建议，但不会修改代码。
- **智能体**：一种自主的编码任务执行模式，具备自我决策、环境感知和工具使用能力。

### 输入要求

选择聊天模式后，在输入框中输入需求描述。建议：

- 结构化你的请求：清晰说明你希望 Qoder 完成的内容
- 提供上下文：包含文件、图片、代码变更等相关信息
- 明确预期：说明任何偏好或规范
- 进行迭代式反馈：针对代码建议或回答提供反馈

## 代码修改与评审

### 多文件编辑

在智能体模式下，Qoder 可能会对多个代码文件进行修改。每项文件修改都包含"生成"和"应用"两个阶段。

### 审阅、接受或拒绝修改

在工作区或单个文件中点击 **查看变更** 按钮以对比修改。然后：

- 使用上下箭头在当前文件中导航并查看变更
- 针对每处变更选择拒绝或接受
- 使用文件级操作区的前进/后退箭头在有变更的文件之间切换

## 工具

Qoder 提供多种工具，协助处理编程中的各类任务：

- 文件搜索
- 文件读取
- 目录读取
- 语义符号搜索
- 文件编辑
- 错误检查
- 命令执行
