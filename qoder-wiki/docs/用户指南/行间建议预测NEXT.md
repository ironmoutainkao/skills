# 行间建议预测（NEXT）

行间建议预测（NEXT）可以基于当前完整代码的上下文，结合代码修改和光标所在位置，动态预测代码变更，让开发者只需 Tab 一下，即可高效完成代码变更。

## 功能特点

借助 Qoder NEXT，你可以：

- 在光标附近一次性编辑多行
- 根据最近的更改和此前接受的编辑获取建议
- 在文件中无缝跳转到下一个建议
- 自动导入依赖项
- 获取跨文件的修改建议，无需手动搜索相关变更

## 工作原理

Qoder 会根据更改代码与 NEXT 提示的宽度，自动以 Inline 或 Side by Side 的方式显示建议：

- 如果二者合计宽度超过编辑器宽度，建议将以 Inline 形式显示
- 否则，将 Side by Side 显示，便于对比

**接受或拒绝建议：**

- 将鼠标悬停在 **接受**/**拒绝** 上，或
- 按 `Tab` 接受，按 `Esc` 拒绝

**预览建议的更改：**

- 按住 `⌥`（Mac）键 / `Alt`（Windows）键预览修改内容
- 松开恢复原始代码

## 设置

### 启用 NEXT

1. 在 Qoder IDE 右上角，点击用户图标，然后选择 **Qoder Settings**
2. 在弹出的面板中，点击 **行间建议**
3. 开启 **NEXT**

### 自动导入

开启后，NEXT 会自动为 TypeScript 导入所需模块

### 快速设置

点击右下角的 NEXT 图标以打开快速设置，你可以：

1. 全局开启或关闭 NEXT
2. 针对特定文件扩展名启用或禁用 NEXT

## 使用场景

- **单文件内的多点预测**：修改变量或函数名时，NEXT 会自动识别同一文件中所有相关的使用位置
- **自动依赖导入**：编写引用外部库或模块的代码时，NEXT 会自动检测并添加所需的 import 语句
- **函数级自动补全**：NEXT 会基于上下文预测整个函数实现
- **跨文件编辑预测**：在一个文件中进行修改时，NEXT 会分析代码库，并主动在其他文件中提出相关修改建议
