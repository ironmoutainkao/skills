# 终端执行异常

## 简介

在使用 Qoder 智能体模式时，终端执行高度依赖于你的本地环境和 shell 配置。你可能会遇到以下问题：

- 无法启动终端
- 命令无法执行
- 没有任何输出

## 常见故障排查方法

### 方法 1：配置受支持的 shell

Qoder 支持多种 shell。请确保你使用的是兼容的 shell。

1. 打开 Qoder
2. 按下 `Cmd + Shift + P`（macOS）或 `Ctrl + Shift + P`（Windows/Linux）打开命令面板
3. 输入 `Terminal: Select Default Profile` 并选择该项
4. 选择一个受支持的 shell：
   - Linux/macOS：`bash`、`fish`、`pwsh`、`zsh`
   - Windows：`Git Bash`、`pwsh`
5. 完全关闭并重新打开 Qoder

### 方法 2：手动安装 shell 集成

在 shell 的配置文件中添加相应语句：

**zsh**（`~/.zshrc`）：
```bash
[[ "$TERM_PROGRAM" == "vscode" ]] && . "$(code --locate-shell-integration-path zsh)"
```

**Bash**（`~/.bashrc`）：
```bash
[[ "$TERM_PROGRAM" == "vscode" ]] && . "$(code --locate-shell-integration-path bash)"
```

**PowerShell**（`$Profile`）：
```powershell
if ($env:TERM_PROGRAM -eq "vscode") { . "$(code --locate-shell-integration-path pwsh)" }
```

## 如果问题仍然存在

1. 点击 **Terminate Terminal** 按钮以关闭当前终端会话
2. 重新运行该命令

## Windows 专属故障排除

### Git Bash

1. 从 https://git-scm.com/downloads/win 下载并安装 Windows 版 Git
2. 退出并重新打开 Qoder
3. 将 Git Bash 设为默认终端

### PowerShell

1. 确保使用 PowerShell 7 或更高版本
2. 调整执行策略：
   ```powershell
   Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

## 异常终端输出

**症状：** 乱码、方块符号、转义序列

**可能原因：** 第三方 shell 个性化配置，例如 Powerlevel10k、Oh My Zsh

**解决方案：** 为 Agent 执行终端禁用复杂提示或主题

```bash
# ~/.zshrc — 当 Qoder Agent 终端执行时禁用 Powerlevel10k
if [[ -n "$QODER_AGENT" ]]; then
  # 跳过主题初始化以获得更好的兼容性
else
  [[ -r ~/.p10k.zsh ]] && source ~/.p10k.zsh
fi
```
