# 因素 IX：衡量重要的事

**规则：追踪目标达成度，而非活动指标。休眠即成功。**

## 虚荣指标 vs 适应性指标

### 虚荣指标（不要追踪这些）

| 指标 | 为什么没用 |
|------|----------|
| 每日消耗的 token 数 | 对产出质量毫无信息量 |
| 会话数量 | 可能 50 个会话修同一个 bug |
| 编写的代码行数 | 更多代码通常是技术债务 |
| 关闭的 Issue 数 | 可能全是简单的，困难的一直拖 |
| 知识库规模 | 10000 条目引用率 0% = 浪费 |
| Agent 在线时间 | 24/7 运行但毫无建树 = 贵的失败 |

**适应性检验：** 这个数字上升了但目标没有推进，你会庆祝吗？如果不会，你在追踪活动不是进步。

### 适应性指标（应该追踪的）

- **目标完成率**：已陈述目标中有多少完全达成了？
- **达成目标所需时间**：从定义到完成多久？
- **复发率**：同类失败出现的频率？（应该下降）
- **知识引用率**：知识库条目有多少被实际使用？
- **干预频率**：人类需要多频繁地拯救系统？
- **每个目标的成本**：达成一个目标的总资源消耗？

## 休眠原则

**最健康的系统是知道何时停止的系统。**

- 目标达成 → 系统停止 ✅
- 没有新的高优先级工作 → 系统等待 ✅
- 目标完成后制造活动 → 这是病态 ❌

设计衡量体系要**奖励休眠**：
- 追踪"目标达成后的空闲时间百分比"（越高越好）
- 追踪"每个达成目标的成本"（越低越好，不鼓励不必要的活动）

## 制造工作螺旋（反面模式）

目标接近完成时出现新工作，以证明持续运营合理性：
- 边缘情况突然变成优先事项
- 重构变得紧急
- "完成"的定义不断偏移

这是活动指标在驱动行为的症状。

## 好的衡量体系示例

```
运营适应性仪表盘（最近 30 天）

目标
  已完成：9 / 12 (75%)
  平均耗时：4.2 天
  每目标成本：$47

学习健康度
  复发问题：3（从 8 下降）
  知识引用次数：127（上升 40%）
  人类干预次数：2（从 11 下降）

系统成熟度
  距上次关键事件：23 天
  空闲时间（目标已达成）：37%
```
