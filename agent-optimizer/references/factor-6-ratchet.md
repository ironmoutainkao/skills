# 因素 VI：锁定前进的进度

**规则：一旦工作通过验证，它就会锁定——不可回退。棘轮只能向前转动。**

## 棘轮模式

```
生成多个并行尝试（廉价）
         ↓
验证过滤（关卡）
         ↓
通过 → 锁定（合并/关闭/记录）
         ↓
失败 → 丢弃（不污染主干）
```

**关键思维转变：**
- 人类开发：最小化浪费，一次做对
- Agent 开发：最小化解决方案时间，并行探索，棘轮过滤

Agent 时间便宜，挂钟时间昂贵。5 个 Agent 并行探索，1 小时出结果，好过 1 个人认真做 8 小时。

## 三个组成部分

**1. 混乱是廉价的（生成多个尝试）**
- 同一任务，不同约束，多个 Agent 并行
- 不协调，竞速，成本可忽略不计
- git worktree 提供免费隔离

**2. 过滤是关卡（验证必须通过）**
- 测试通过（单元/集成/端到端）
- 代码检查通过
- PR 审查批准
- 验证是布尔值：通过或不通过，没有"基本完成"

**3. 棘轮是锁（通过即永久）**
```bash
# 受保护分支配置
main branch protection:
  - Require PR approval
  - Require CI status checks
  - No force pushes
  - No deletions
```

## 失败预算机制

- 设定重试上限（如 3 次）
- 耗尽后：升级，而不是继续重试
- 成功的尝试立即锁定，无法撤销

没有棘轮 → 无限折腾（Agent 可以永远重试同一个失败方法）

## 完成记录账本

每一次合并、关闭、验证成功都记录：
```json
{
  "timestamp": "2026-04-09T00:30:00+08:00",
  "agent": "agent-alpha",
  "action": "merge",
  "issue": "auth-456",
  "tests_passed": true,
  "review_approved": true
}
```

这是证据链，也是能力积累的记录。

## 反面模式：没有棘轮的反复折腾

Agent 改了 → CI 失败 → Agent 回退 → 再改 → CI 以另一种方式失败 → 无限循环

没有棘轮，没有向成功推进的强制力。有了棘轮：失败预算耗尽 → 升级给人类 → 决策 → 继续。
